Как это работает

n8n отправляет событие в /api/incoming/n8n, а правило определяет, кому и в каком виде уйдёт сообщение. Можно использовать rule_key или прямых получателей в payload.

Минимальный payload

{"rule_key":"incident-critical","message":"Падает billing","request_id":"evt-001"}

Сейчас правила можно использовать для входящих событий из n8n и Alertmanager.

Последние входящие события

Последние вызовы inbound API для быстрой проверки срабатываний и ошибок.
